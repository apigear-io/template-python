# Contributing

Thanks for considering a contribution to the ApiGear Python SDK template.

This repo is a code-generation template — its purpose is to produce Python
SDKs from ApiGear interface descriptions, not to run as an application
itself. Most changes therefore touch the `templates/` directory and are
verified by regenerating the `goldenmaster/` reference output and running
its test suite.

## Repository layout

| Path | Purpose |
|---|---|
| `templates/` | Go-text-template files the apigear CLI consumes. The source of truth for what gets generated. |
| `rules.yaml` | Declares which features exist, their dependencies, and where each template file lands in a generated project. |
| `goldenmaster/` | Checked-in reference output of `templates/` against `apigear/test-apis/`. CI diffs newly-generated output against this. |
| `test/` | Local-only working copy regenerated by `task generate_solution`; gitignored output target. |
| `apigear/test-apis/` | Submodule of the canonical ApiGear test interface set. |
| `docs/` | Docusaurus pages published on the ApiGear documentation site. |

## Setup

```bash
# install the pinned apigear CLI and dev tools
task install

# regenerate test/ from templates and diff against goldenmaster
task generate_solution
task diff

# install + run the goldenmaster test suite (requires a local mosquitto)
task py:install_goldenmaster
task py:test_goldenmaster
```

You will need:

- Python ≥ 3.11
- Go (for `task install` to build the `apigear` CLI)
- Mosquitto running on `localhost:1883` for the MQTT integration tests
  (`apt install mosquitto`, `brew install mosquitto`, or
  `docker run -p 1883:1883 eclipse-mosquitto`)

## Workflow for changing a template

1. Edit files under `templates/`.
2. Run `task generate_solution` — this regenerates `test/`.
3. Run `task diff` — confirms the new output is what you intended.
4. If the diff is intentional, sync `test/` over `goldenmaster/`:
   `cp -r test/. goldenmaster/`.
5. Run `task py:test_goldenmaster` to verify the regenerated SDK still
   passes its tests.
6. Commit both the template change and the goldenmaster update in the same
   commit.

PRs that modify a template without updating the goldenmaster will fail
`task diff` in CI.

## Commit messages

This repo follows [Conventional Commits](https://conventionalcommits.org).
Examples from the recent history:

- `feat(mqtt): expose subscription id to adapters`
- `fix(olink): fix generated code for arrays`
- `docs: close v1.1.0 documentation gaps`
- `ci: refresh workflows and pin tooling versions`
- `test(mqtt): clear retained messages at pytest session start`

Write the message about what the change does and why, not about its place
in a planning bucket — those don't survive in `git log`.

## Reporting bugs / requesting features

- Bugs and feature requests:
  <https://github.com/apigear-io/template-python/issues>
- Discussion:
  <https://github.com/orgs/apigear-io/discussions>

If you're filing a bug, please include the `apigear` CLI version
(`apigear version`), the Python version, and the contents of your
`solution.yaml` / `module.yaml` if the bug is generation-time.

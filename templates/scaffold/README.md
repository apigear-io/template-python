# Python API

This project was generated by the [ApiGear](https://apigear.io) Python template. It contains pydantic data types, abstract interfaces, a runnable implementation skeleton you can edit, and — depending on which features you selected — HTTP, ObjectLink, and MQTT adapters.

## Requirements

- **Python ≥ 3.11**
- `pip` (recent version recommended)

> Note: it is recommended to install the dependencies in a virtual environment (`venv`).

## Install dependencies

```bash
pip install --upgrade pip
pip install --upgrade -r requirements.txt
```

The exact contents of `requirements.txt` depend on which features were enabled at generation time (for example, `paho-mqtt` is only listed when the `mqtt` feature was selected).


## Implement your services

The generated abstract interfaces live in `{module_name}/api/api.py` as `I<Interface>` classes. Default-behavior implementations are scaffolded in `{module_name}/impl/<interface>.py`. Each generated implementation:

- Subclasses the matching `I<Interface>`.
- Stores property values on the instance and exposes `get_*` / `set_*` accessors.
- Fires `EventHook` events on property change and on signal emission.
- Returns type-correct defaults from operations.

Edit those `impl/` files to add real behavior. The generator marks them with `preserve: true`, so re-running generation will not overwrite your edits.

## Run a service

Which entry point you use depends on the features you generated:

- **`http`** — run the FastAPI app:

  ```bash
  uvicorn http.server:app --host 0.0.0.0 --port 8000
  ```

  See the `{module_name}/http/` folder for routes and the sync client.

- **`olink`** — see `examples/olink/server.py` for a working server and `examples/olink/client.py` for a client. A short walk-through is in `examples/olink/OLINK_README.md` (only generated when `olink` is enabled).

- **`mqtt`** — see `examples/mqtt/server.py` and `examples/mqtt/client.py`. You will need to run an MQTT broker of your choice; the broker is not bundled with this project.

## Tests

If `test_helpers` was enabled, reusable builders are generated under `{module_name}/test_helpers/`. The `scaffold` feature also drops a pytest skeleton (`{module_name}/impl/test_<interface>.py`) per interface; run with:

```bash
pytest
```
